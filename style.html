Style guide — Timeline / Gantt app

Live reference for the Things 3-style theme. Use this when restyling this app or starting a new one in the same family. Copy the :root block from this file's <style>, then copy individual component rules below. Always consume var(--…) tokens — never hardcode hex.

See also STYLE.md for the short written rules & anti-patterns.

12. Interaction states (selection / hover / focus)

Use these consistently. No outline:2px solid, no glow, no neon ring.

/* Selection on a row    */ background:var(--accent-dim); color:var(--accent);
                            /* optional left bar: */ box-shadow:inset 3px 0 0 var(--accent);
/* Hover on a row        */ background:rgba(0,0,0,.03);
/* Focus on an input     */ border-color:var(--accent); box-shadow:0 0 0 3px var(--accent-dim);
/* Active in segmented   */ background:#ffffff; color:var(--accent); box-shadow:0 1px 3px rgba(0,0,0,.08);

13. Anti-patterns — do NOT do these

Hardcoded dark hexes — #0c1c19, #021412, #05100e survived from the old dark theme and looked broken on white. Always use var(--…).
Dark surface with 2px corners
Glow / neon shadows — they belong to the Aurora backup theme, not Things.
Glowing button
White text on a light background — the original .calendar-cell.today .cv-date-num: color:#fff bug. Always check contrast.
19
Saturated brand-style status pills — full-strength #ef4444 red / #22c55e green look harsh. Use the 12–20% opacity recipe in section 7.
Sharp 2–4px radii on anything. Pick from 8 / 12 / 14 / 999.
Inline style={{…}} colour values in JSX — defeats the token system. Add a class and use var(--…).

14. Adding a new component — checklist

  1. Read STYLE.md. Skim this file's sections 1–3.
  2. Use var(--…) tokens only. If you need a colour that doesn't exist, add a new token to the Things theme block in timeline.html first.
  3. Match an existing component's radius, padding, and hover treatment. Find the closest analogue above and mirror it.
  4. Scope theme overrides under body[data-theme="things"] in timeline.html so the Aurora backup and any future themes aren't polluted.
  5. Verify visually. Load the preview, open the sample project, and confirm the new component looks at home next to the Welcome / List / Calendar / Gantt views.
